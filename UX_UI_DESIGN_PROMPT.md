# UX/UI Redesign Prompt — RodBus NFC Tracker

> ใช้ Prompt นี้กับ AI Design Tools (เช่น Figma AI, v0.dev, Claude, GPT) เพื่อให้ออกแบบ UX/UI ใหม่ทั้งระบบ

## 1. ภาพรวมโปรเจค (Project Overview)

**RodBus NFC Tracker** คือแอปพลิเคชันแบ่งค่าใช้จ่ายรถตู้/รถร่วม สำหรับคนขับรถและผู้โดยสาร ระบบทำงานผ่าน NFC Tap หรือ QR Code Scan เพื่อบันทึกการเดินทาง แล้วคำนวณค่าใช้จ่ายอัตโนมัติ แบ่งอย่างยุติธรรม

**Tech Stack:** Next.js 15, React 19, TypeScript, Tailwind CSS 4, Clerk Auth, PostgreSQL (Neon), Prisma ORM
**ภาษา:** รองรับ 2 ภาษา (ไทย / อังกฤษ) พร้อมปฏิทินพุทธศักราช
**Target Users:** คนขับรถตู้ไทย และผู้โดยสารประจำ (ใช้งานบนมือถือเป็นหลัก)

---

## 2. บทบาทผู้ใช้ (User Roles)

| Role | สิทธิ์การใช้งาน |
|------|----------------|
| **PENDING** | ผู้ใช้ใหม่รอ Admin อนุมัติ — เห็นแค่หน้ารออนุมัติ |
| **USER (ผู้โดยสาร)** | Tap NFC/QR เช็คอิน, ดูยอดหนี้ค้างชำระ, ดูประวัติเดินทาง/ชำระเงิน |
| **ADMIN (คนขับ/เจ้าของรถ)** | ทุกอย่างของ USER + บันทึกค่าน้ำมัน/ที่จอดรถ, จัดการหนี้, จัดการผู้ใช้, จัดการรถ, ปิดวันที่ทำการ, สร้าง QR Code |

## 3. Data Models (โครงสร้างข้อมูล)

```
User {
  id, clerkId, name, email, image, role (PENDING/USER/ADMIN)
  → ownedCars[], trips[], payments[]
}

Car {
  id, name, licensePlate, defaultGasCost, ownerId
  → owner(User), trips[], tripCosts[], payments[]
}

Trip {                          // 1 record = 1 การแตะ NFC ของผู้โดยสาร 1 คน
  id, userId, carId, tripCostId?, date, tappedAt
}

TripCost {                      // ค่าใช้จ่ายรวมรายวันต่อรถ
  id, carId, date, gasCost, parkingCost, label
  → trips[]
}

Payment {                       // บันทึกการชำระหนี้
  id, userId, carId, amount, note, date
}

DisabledDate {                  // วันหยุดที่ปิดระบบ
  id, date, reason
}
```

## 4. ระบบการคำนวณ (Calculation Logic) — ออกแบบ UI ให้แสดงผลทุกขั้นตอนอย่างชัดเจน

### 4.1 การคำนวณค่าใช้จ่ายต่อหัว

```
สูตรหลัก:
─────────────────────────────────────────────────
headcount = จำนวนผู้โดยสารที่ไม่ซ้ำ + 1 คนขับ (ถ้าคนขับไม่ได้อยู่ในรายชื่อผู้โดยสาร)

ค่ารวมต่อหัว   = (ค่าน้ำมัน + ค่าที่จอดรถ) ÷ headcount
ค่าน้ำมันต่อหัว = ค่าน้ำมัน ÷ headcount
ค่าจอดรถต่อหัว  = ค่าที่จอดรถ ÷ headcount
─────────────────────────────────────────────────

ข้อยกเว้น: คนขับ (เจ้าของรถ) ไม่ต้องจ่าย — ส่วนของคนขับถูกกระจายให้ผู้โดยสารทั้งหมด
ปัดเศษ: ทุกค่าปัดเป็นทศนิยม 2 ตำแหน่ง → Math.round(value × 100) / 100
```

### 4.2 การจ่ายหนี้ (Payment Application — FIFO)

```
ลำดับการหักหนี้:
─────────────────────────────────────────────────
1. เรียงหนี้ตามวันที่ (เก่าสุดก่อน)
2. นำยอดชำระไปหักจากหนี้เก่าที่สุดก่อน
3. ถ้าชำระเกินหนี้ก้อนนั้น → ส่วนเกินไปหักก้อนถัดไป
4. ถ้าชำระไม่ครบ → หนี้ก้อนนั้นเหลือเท่าที่ยังไม่ได้จ่าย

การคำนวณ partial payment ratio:
  adjusted_share = entry.share × (1 - remaining_payment / entry.share)
─────────────────────────────────────────────────
```

### 4.3 ตัวอย่างการคำนวณ (ใช้สำหรับ Design แสดงผล)

```
ตัวอย่าง: วันที่ 10 มี.ค. 2569
─────────────────────────────────────────────────
รถ: Toyota HiAce (กก-1234)
ผู้โดยสาร: สมชาย, สมหญิง, สมศรี (3 คน)
คนขับ (เจ้าของรถ): สมบัติ

headcount = 3 ผู้โดยสาร + 1 คนขับ = 4 คน

ค่าน้ำมัน:  ฿200
ค่าจอดรถ:  ฿100
ค่ารวม:    ฿300

ค่าต่อหัว:        300 ÷ 4 = ฿75.00
  ├─ ค่าน้ำมัน:   200 ÷ 4 = ฿50.00
  └─ ค่าจอดรถ:   100 ÷ 4 = ฿25.00

สมบัติ (คนขับ): ไม่ต้องจ่าย ✓
สมชาย:  หนี้ ฿75.00
สมหญิง: หนี้ ฿75.00
สมศรี:  หนี้ ฿75.00
─────────────────────────────────────────────────

ตัวอย่าง partial payment:
สมชายจ่าย ฿100 → หนี้วันที่ 10 (฿75) หมด, เหลือ ฿25 → ไปหักหนี้วันถัดไป
```

## 5. หน้าจอทั้งหมดที่ต้องออกแบบ (All Screens)

### 5.1 🏠 Dashboard (หน้าหลัก)

**สำหรับ USER (ผู้โดยสาร):**
- **การ์ดยอดหนี้รวม** — แสดงยอดหนี้ค้างชำระของเดือนปัจจุบัน เป็นตัวเลขใหญ่สีแดง (฿XXX.XX)
- **รายละเอียดหนี้ (Expandable Breakdown)** — กดเปิดดูรายละเอียด:
  - แต่ละรายการแสดง: ชื่อรถ, วันที่, ยอดหนี้
  - เปิดรายละเอียดย่อย:
    - `ค่าน้ำมัน ฿200 ÷ 4 คน = ฿50.00`
    - `ค่าจอดรถ ฿100 ÷ 4 คน = ฿25.00`
  - แสดงทีละ 5 รายการ + ปุ่ม "ดูเพิ่มเติม"
- **เที่ยวเดินทางล่าสุด** — 5 รายการล่าสุด แสดง: วันที่, เวลา Tap, ชื่อรถ, เที่ยวที่

**เพิ่มเติมสำหรับ ADMIN (คนขับ):**
- **ฟอร์มบันทึกค่าใช้จ่าย:**
  - Dropdown เลือกรถ (ถ้ามีหลายคัน)
  - ช่องกรอกค่าน้ำมัน (pre-fill จาก defaultGasCost ของรถ)
  - ช่องกรอกค่าจอดรถ (ค่าเริ่มต้น 0)
  - ปุ่ม "บันทึก" พร้อม loading state
- **ส่วนจัดการหนี้ (Debt Settlement):**
  - รายชื่อผู้ใช้ที่มีหนี้ค้าง เรียงจากมากไปน้อย
  - แต่ละคนแสดง: ชื่อ, ยอดหนี้ค้าง, ปุ่ม "ชำระแล้ว"
  - กดดู Breakdown ของแต่ละคน
  - Confirm dialog ก่อน Mark as Settled

---

### 5.2 📜 History (ประวัติ) — 3 แท็บ

**Tab 1: เที่ยวเดินทาง (Trips)**
- ตัวกรองวันที่ (เริ่มต้น — สิ้นสุด)
- Infinite scroll (ครั้งละ 10 รายการ)
- จัดกลุ่มตามวันที่ + นับเที่ยว
- แสดง: วันที่, เวลา Tap, ชื่อรถ, เที่ยวที่
- Admin: แก้ไข/ลบ timestamp ได้

**Tab 2: การชำระเงิน (Payments)**
- ตัวกรองวันที่
- Infinite scroll
- แสดง: ชื่อรถ, วันที่ชำระ, จำนวนเงิน, หมายเหตุ
- Expandable details

**Tab 3: สรุป (Summary)**
- เลือกช่วง: วัน / เดือน / ปี
- จัดกลุ่มตามช่วงที่เลือก
- แต่ละกลุ่มแสดง:
  - สถานะ: ✅ ชำระแล้ว / ⏳ ค้างชำระ
  - ยอดสะสม / ยอดจ่าย / ยอดค้าง per user
  - ค่าน้ำมันรวม / ค่าจอดรถรวม
- **Hierarchical drill-down:** ปี → เดือน → วัน → รายรถ
- แต่ละระดับแสดงสูตรการคำนวณ:
  - `ค่าน้ำมัน ฿XXX ÷ X คน = ฿XX.XX`
  - `ค่าจอดรถ ฿XXX ÷ X คน = ฿XX.XX`

### 5.3 ✅ Tap Confirmation & Success

**หน้ายืนยัน Tap (tap-confirm):**
- แสดงชื่อรถที่กำลังจะ Tap
- ปุ่มยืนยัน / ยกเลิก

**หน้าผลลัพธ์ Tap (tap-success):**
- สถานะต่างๆ ที่ต้องออกแบบ:
  - ✅ **เช็คอินสำเร็จ** — "บันทึกเรียบร้อย เที่ยวที่ X"
  - ⚠️ **บันทึกแล้ว** — "คุณได้เช็คอินวันนี้แล้ว" (anti-fraud 2 ชม.)
  - ❌ **ไม่มี Trip เปิดอยู่** — "ยังไม่มีเที่ยวเปิดสำหรับวันนี้"
  - 🚫 **ระบบปิด** — "วันนี้ระบบไม่เปิดให้บริการ"
  - 🔒 **เจ้าของรถ Tap ตัวเอง** — "เจ้าของรถไม่ต้อง Tap"

### 5.4 🔧 Admin Panel

**5.4.1 จัดการผู้ใช้ (User Management):**
- รายชื่อผู้ใช้ทั้งหมด
- Badge สถานะ: 🟡 PENDING / 🟢 USER / 🔴 ADMIN
- Actions: อนุมัติ / เปลี่ยน Role / ลบออก
- ผู้ใช้ PENDING แสดงปุ่ม "อนุมัติ" เด่นชัด

**5.4.2 จัดการรถ (Car Management):**
- รายชื่อรถ: ชื่อ, ทะเบียน, ค่าน้ำมันเริ่มต้น
- เพิ่มรถใหม่ / ลบรถ
- ตั้งค่า Default Gas Cost

**5.4.3 จัดการวันที่ (Date Management):**
- ปฏิทินแสดงวันที่ปิดทำการ
- เพิ่ม/ลบวันปิด + เหตุผล (เช่น "วันหยุดนักขัตฤกษ์")

**5.4.4 QR Code:**
- แสดง QR Code ของแต่ละรถ
- URL ที่ลิงก์ไปหน้า Tap
- ดาวน์โหลด/แชร์ QR

## 6. UI Design Requirements (ข้อกำหนดการออกแบบ)

### 6.1 Design System

```
Primary Color:    Blue (#2563EB / blue-600) — ปุ่มหลัก, ลิงก์
Success Color:    Green (#16A34A / green-600) — ชำระแล้ว, สำเร็จ
Danger Color:     Red (#DC2626 / red-600) — หนี้, ข้อผิดพลาด, ลบ
Warning Color:    Yellow/Amber (#D97706) — รอดำเนินการ, คำเตือน
Neutral:          Slate/Gray — ข้อความรอง, พื้นหลัง

Background:       Gradient (slate-50 → white → blue-50/40)
Cards:            White, rounded-xl (12px) ถึง rounded-2xl (16px), shadow เบาๆ
Font:             Inter (sans-serif)
Currency Format:  ฿XXX.XX (Thai Baht)
Calendar:         พุทธศักราช (พ.ศ.) สำหรับภาษาไทย
```

### 6.2 Responsive Design

```
Mobile First (ความสำคัญสูงสุด):
  - ผู้ใช้ส่วนใหญ่ใช้มือถือ Tap NFC
  - Card-based layout
  - Touch-friendly targets (min 44px)
  - Bottom navigation หรือ hamburger menu

Tablet:
  - 2-column layout สำหรับ Admin
  - Side panel สำหรับ detail views

Desktop:
  - Full dashboard layout
  - Tables แทน cards สำหรับข้อมูลเยอะ
  - Side-by-side comparison views
```

### 6.3 Interaction Patterns

```
Expandable/Collapsible:  รายละเอียดหนี้, สูตรคำนวณ
Infinite Scroll:          ประวัติเดินทาง, การชำระเงิน
Load More Button:         รายการ Breakdown (ทีละ 5)
Confirmation Dialog:      ก่อน Mark as Settled, ลบรถ, ลบผู้ใช้
Loading States:           ปุ่มแสดง "..." ขณะโหลด + disable
Animations:               fade-in, scale-in สำหรับ cards
Pull to Refresh:          สำหรับ mobile dashboard
```

### 6.4 Data Visualization Requirements

```
ยอดหนี้หลัก:     ตัวเลขขนาดใหญ่ สีแดงเด่นชัด พร้อมสัญลักษณ์ ฿
สูตรคำนวณ:       แสดงเป็นบรรทัดย่อย เช่น "฿200 ÷ 4 คน = ฿50.00"
Headcount:        แสดงจำนวนคนที่ร่วมเดินทาง "4 คน"
สถานะการชำระ:    Badge สี (เขียว=ชำระแล้ว, แดง=ค้างชำระ, เหลือง=รอ)
Progress:         แถบแสดง % ที่ชำระแล้ว vs ค้างชำระ (optional)
Timeline:         แสดงประวัติเรียงตามวันที่
Drill-down:       ปี → เดือน → วัน → รายละเอียดต่อรถ
```

### 8.1 Dashboard Widgets (Optional)
- **กราฟแนวโน้ม:** กราฟเส้นแสดงค่าใช้จ่ายรายเดือน
- **Pie Chart:** สัดส่วน ค่าน้ำมัน vs ค่าจอดรถ

## 9. Anti-Fraud & Edge Cases (ออกแบบ UI ให้รองรับ)

```
- Debounce 2 ชม.:     แสดงข้อความ "คุณได้ Tap ไปแล้ว กรุณารอ 2 ชั่วโมง"
- เจ้าของรถ Tap ตัวเอง: แสดงข้อความ "เจ้าของรถไม่จำเป็นต้อง Tap"
- วันปิดทำการ:        แสดงข้อความ + เหตุผล "วันนี้หยุดทำการ: วันหยุดนักขัตฤกษ์"
- ไม่มี TripCost:     แสดงข้อความ "ยังไม่มีค่าใช้จ่ายสำหรับวันนี้"
- หนี้ = 0:           แสดงข้อความเชิงบวก "ไม่มีหนี้ค้างชำระ 🎉"
- ผู้ใช้ใหม่ PENDING:  หน้ารอพร้อมอธิบายขั้นตอน
```
